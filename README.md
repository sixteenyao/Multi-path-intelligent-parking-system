# Smart Camera — RK3588 端侧 AI 安防摄像头（LLM 增强版）

## 一句话定位

> 不只是"检测到人了"，而是"一个穿红色外套的快递员在下午3点15分把包裹放在了门口"——在 RK3588 上端侧实现的视觉理解 + 自然语言描述系统。

## 为什么这个项目有区分度

```
传统安防摄像头:  有人 → 告警                    (烂大街)
本系统:          有人 → 检测+裁剪 → LLM描述 → "穿红色外套的人放下包裹后离开"
                 支持自然语言查询 "今天来了几次快递？"
                 所有处理在 RK3588 端侧完成，不上云
```

## 核心卖点（面试抓手）

### 1. 端侧 LLM 推理
- RK3588 的 RKLLM 工具链可以跑 Qwen2.5-0.5B / 1.5B
- 模型放在 NPU 上推理，不占 CPU
- 关键帧才触发 LLM（不是每帧），省算力

### 2. 三层智能

| 层级 | 做什么 | 延迟 | 触发频率 |
|------|--------|------|----------|
| 快速通道 | YOLO 检测 person/car/package | ~30ms | 每帧 |
| 深度通道 | VLM 场景描述（谁、在做什么） | ~500ms | 关键事件 |
| 查询通道 | 自然语言查询历史事件 | ~1s | 用户主动 |

### 3. 隐私优先
- 原始视频不出设备
- 只推结构化描述或模糊化后的视频
- 查询在本地执行，不经过云端

## LLM 触发策略（关键设计）

不是每帧都跑 LLM，RK3588 扛不住：

```
YOLO 检测到 person (置信度 > 0.8)
  → 裁剪目标区域
  → 判断是否需要 LLM:
      ✓ 新目标出现（track_id 首次出现）
      ✓ 目标行为变化（进入/离开/放下物品）
      ✓ 用户主动查询
      ✗ 同一目标持续存在（跳过，YOLO 够了）
  → 把裁剪图 + 提示词送给 VLM
  → 输出: "一个穿着深色衣服的人站在门口，手里拿着包裹"
```

## 构建

```bash
# 在 RK3588 上
# 依赖: librknn_api.so, librkllm_api.so, librockchip_mpp.so, librga.so
#       OpenCV, FFmpeg, yaml-cpp
mkdir build && cd build
cmake .. && make -j$(nproc)

# 运行
./smart_camera --config ../config/cameras.yaml
```
