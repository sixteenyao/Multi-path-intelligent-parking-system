# 基于 AI 视觉 + 大模型 + 云存储的智能分布式停车场监控系统

> 部署于 RK3588 嵌入式平台的轻量级停车场智能化升级方案 —— 无需替换存量摄像头硬件，即可实现区域管控、车辆检测、进出记录与违停告警。

**开发周期**：2026.6 – 2026.7

---

## 项目描述

AI 时代，针对停车场大存量摄像头智能化不足的痛点，本项目开发了一套轻量级智能升级方案：**无需替换硬件即可实现区域管控、车辆检测、进出记录与违停告警**等功能。

项目部署于 **RK3588** 嵌入式平台，基于 **DRM/KMS** 实现显示输出，适用于多区域同步监控场景。系统通过提前设定的停车场及车位区域蒙版，结合 **YOLO11** 算法实现车辆的精准检测与违停识别，并引入**目标跟踪**机制，对车辆进出状态进行实时判断。

同时，系统支持日志与流媒体数据的**云端存储**，采用**断点续传**策略确保传输可靠性和数据持久化。此外，将云端日志数据用于 **RAG 知识库**构建，借助 **qwen-vl-plus** 多模态大模型以及 **qwen-plus** 文本大模型能力，使管理员能够通过**自然语言问答**快速掌握某一停车场区域的整体情况，无需逐一查看监控画面，提升管理效率。

## 功能界面

系统以触摸界面呈现多路监控通道，每个通道均支持在实时画面上叠加 **检测框 / 停车场蒙版 / 车位蒙版**（可分别开关），并将车辆进出、违停事件实时写入日志。

| 通道 | 输入来源 | 说明 |
|------|----------|------|
| A – D | 本地监控视频 | 多区域停车场真实监控场景回放，四路后台原速播放并检测 |
| E | RTSP 网络流 | 网络摄像头 / 手机推流接入（FFmpeg 低延迟 TCP 传输） |
| F | MIPI CSI 摄像头 | 板载 MIPI CSI 摄像头实时接入（V4L2 原生，复用区域蒙版做检测与日志） |

> 所有通道**无论是否处于前台显示，均在后台实时抓帧与检测**，进出 / 违停事件持续记录。

## 主要功能与亮点

1. **实时区域化车辆检测**：实现实时蒙版车辆检测、进出检测及违停检测；通过画质增强、蒙版优化与算法优化，综合准确率从原有 **70% 提升至 88%**。

2. **多源视频接入**：支持多种视频流接入方式，涵盖 **RTSP 网络流、HDMI 采集、MIPI CSI 摄像头、USB 摄像头及本地视频文件**。

3. **可靠的云端归档**：采用**定时上传 + 断点续传**机制，联网时每 60 秒向云端传送数据，断网时数据本地保存、联网后自动补传，上传完整率达 **99%**。

4. **极致轻量化**：动态链接编译后二进制文件约 **100KB**；静态链接裁剪编译后体积约 **70MB**（含库文件、YOLO 模型、字体与提示音资源）。

5. **多模态大模型 + RAG**：集成多模态大模型进行车辆特征识别，构建 RAG 知识库以实现自然语言日志查询；通过**异步非阻塞调用**，响应速度约 **1.5 ~ 4s**（多模态查询基本 > 3s）。

6. **NPU 加速推理**：采用 NPU 加速 YOLO11s（INT8 量化）模型推理，相比常规 YOLO11m，单帧推理耗时从 **100+ ms 降至 60ms 以内**，内存与体积占用降低约 **50%**。

## 构建与运行

依赖：`librknnrt.so` / `librknn_api.so`、`librockchip_mpp.so`、`librga.so`、OpenCV、FFmpeg、FreeType、yaml-cpp。

```bash
# 方式一：板子本地编译（RK3588 上）
mkdir build && cd build
cmake .. && make -j$(nproc)

# 方式二：交叉编译（x86 → RK3588）
mkdir build && cd build
cmake .. -DCMAKE_TOOLCHAIN_FILE=../cmake/toolchain.rk3588.cmake
make -j$(nproc)
```

在 RK3588 上运行 GUI 程序（需要 DRM 显示与触摸设备）：

```bash
./smart_camera_gui
```
